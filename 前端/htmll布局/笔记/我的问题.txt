3.3 清除浮动方法
1. 额外标签法也称为隔墙法,是W3C推荐的做法。(了解)
    额外标签法会在浮动元素末尾添加一个空的标签(必须块级元素)。例如<div style="clear:both"></div>,或者其他标签
    (如<br/>等)。
2. 父级添加overflow 属性
    可以给父级添加overflow 属性,将其属性值设置为hidden、auto或scroll。
3. 父级添加after伪元素
:after方式是额外标签法的升级版。也是给父元素添加
clearfix:after {
    content: "";
    display: block;
    height: 0;
    clear: both;
    visibility: hidden;
}

4. 父级添加双伪元素
.clearfix:before, .clearfix:after {
    content:"";
    display:table;}
.clearfix:after {clear:both;}

4.其他属性(CSS3):content/cursor/border-radius/box-shadow/text-shadow/background:linear-gradient ...

浮动的元素会压住其他盒子，并只对在浮动元素之后的盒子生效，具体效果是后面的盒子向前占位(占浮动元素之前的位置)
并且浮动的元素会压住这个未浮动的盒子，若二者都浮动，则无间隙的按顺序排列，并且浮动的元素会具有行内块元素的特性.
设置浮动后，后面的元素会占据浮动元素的位置，导致浮动的元素会遮挡住后面的元素，但是内容不会被遮挡，如图片、文字。
浮动的元素脱离文档流后不再占据位置,要像让已经脱离文档流浮动的元素再次占据原先位置就需要在父元素清除浮动

在相对定位和绝对定位中，由于相对定位未脱离文档流，所以会保留一些元素原本的特性，比如块元素div仍然独自占据一行,同时根据盒子内元素的大小来撑开盒子
同时相对定位在原位置 会遮盖住脱离文档流的元素和标准流
而绝对定位已经脱离文档流，会使 ——>比如div根据盒子内的元素的大小撑开相应的大小，包括宽度，即不再肚子占据一行
同时若绝对定位的元素再一个普通文档流元素下方，则会先按照正常排列，但是在实际写样式的过程中并无影响

1. transform（变换）
定义：transform 是一个CSS属性，用于直接改变元素的几何形态（如位置、大小、角度），但它本身不产生动画效果。它支持多种函数，包括translate、rotate、scale等。
用途：
    移动元素位置（通过translate函数）。
    旋转元素（如rotate(90deg)）。
    缩放元素大小（如scale(1.2)）。
    这些改变会立即生效，不会自动添加过渡效果。
示例：transform: translate(50px, 100px); 会将元素向右移动50px，向下移动100px。
特点：
    实际改变元素的视觉效果。
    不是动画控制器，只是定义最终状态。
    常用于布局调整或静态变换。
引用中提到：transform 用于“实际改变元素形状、位置等”，但“不会自动产生动画”1。
2. translate（平移）
定义：translate 不是独立的CSS属性，而是transform属性的一个具体函数。它专门用于移动元素的位置。
用途：
    在水平或垂直方向上平移元素（移动距离基于坐标系）。
    通常作为transform的一部分使用，例如transform: translate(20px, 0);。
    示例：transform: translateX(30px); 使元素沿X轴移动30px。
特点：
    是transform的子集，不能单独存在。
    只负责位置移动，不涉及旋转或缩放。
    与其他变换函数（如rotate）组合使用更高效。
    引用中提到：translate常用于“定义元素的变换操作”，但需配合transition实现动画3。
3. transition（过渡
定义：transition 是一个CSS属性，用于控制CSS属性（如transform）变化时的动画过渡效果。它是一个简写属性，包含持续时间（transition-duration）、延迟（transition-delay）等参数。
用途：
    平滑过渡属性变化（如颜色渐变、位置移动）。
    定义动画的速度曲线（如ease-in）。
    需要属性值变化（如hover或JS事件）才能触发。
示例：transition: transform 0.5s ease; 表示当transform变化时，用0.5秒平滑过渡。
特点：
    是动画控制器，自身不改变元素，只管理变换过程。
    常用于transform、opacity、color等属性。
    能增强用户体验，避免突变效果。
    引用中提到：transition 是“动画属性”，决定“变换过程是否平滑”
